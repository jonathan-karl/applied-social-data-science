  1. The goal of this part of the assignment is to obtain country-level development indicators that may be related to linguistic fragmentation from the World Bank API. You can read the documentation and instructions here.

Write your own function that will take an indicator code (e.g. SP.POP.TOTL) as input, query the API, parse the JSON output into R, and then return a clean data frame where each row is a country. If you’re not able to figure it out, you can also use the WDI package. Feel free to take a look at the code for some clues on how to query the API (see e.g. here). (15 points)

indicator_function is created, which enables the user to type in an indicator code in the subsequent ‘update_url’ feature. This will update the relevant part of ‘url’ with the indicator code (SP.POP.TOTL is used here as an example). In this example, the url will take all country/region data (264) in 2019 from the World Bank website.

The API is queried and the JSON output is parsed into R with fromJSON(). The output is stored in a tibble data frame called ‘country_population’, which is then cleaned. Notably, three empty columns are removed, and ‘countryiso3code’ column is renamed to ‘country.code’. Countries with NA indicator values are omitted. A sample of the data is then printed using tail(). Pipe operators (%>%) facilitate this.

indicator_function <- function(indicator_id) {
  #' Function runs over the API link and replaces indicator_id with relevant input from update_url
  url <- sprintf(paste0("http://api.worldbank.org/v2/country/all/indicator/", indicator_id, "?format=json&per_page=264&date=2019"))
}

# Update url with an indicator code, e.g. NY.GDP.PCAP.KD if GDP per capita indicator is of interest:
update_url <- indicator_function('SP.POP.TOTL') # Population indicator_id

# Takes URL from WB API, parses into R, converts to tibble, reformats, omits NA's
country_population <- fromJSON(update_url) %>%
  pluck(2) %>%
  as_tibble() %>%
  rename(country.code = countryiso3code) %>%
  select(-unit, -obs_status, -decimal) %>%
  na.omit

  1. Using the function you just created, get country-level data on two metrics: GDP per capita and another metric that you consider could be correlated with language fractionalization (see the Alesina et al paper for inspiration). (5 points)

Country-level data on GDP per capita (indicator code: NY.GDP.PCAP.CD) and received remittances (indicator code: BX.TRF.PWKR.CD.DT) is achieved by using the indicator_function defined in question 1. The data is retrieved and stored in two tibbles (‘country_GDP’ and ‘country_remittances’ respectively) using similar methodology to question 1 (i.e. the same cleaning process takes place). Snippets of the tibbles are printed with tail().

indicator_vec represents a vector of the two indicator codes which allows the lapply function to be applied. This means that the code does not need to be repeated and instead it will loop through the indicator ids in indicator_vec to create tibbles for each indicator.

Remittances are transfers from one country to another, usually by international workers to individuals in their country of origin. This could be correlated with language fractionalization if countries with multiple national languages (e.g. Switzerland) attract workers with those language skills from a wider pool of countries (e.g. German, French and Italian speaking countries), who could then send money back to their home countries.

indicator_vec <- c('NY.GDP.PCAP.CD', 'BX.TRF.PWKR.CD.DT')

indicator_list <- lapply(indicator_vec, function(x) {

The findings of Alesina et al. (2003) deviate strongly from what our analysis of the relationship between GDP per capita and language fractionalization found. Whereas, Alesina et al. postulated a negative relationship between the two variables in their paper (and we were able to reproduce this result in a simple and uncontrolled linear regression), our analysis with Twitter-data found the opposite relationship, which was in fact more positive than the relationship between the data from Alesina et al. (2003) was negative. Although this finding appears striking at first sight, this indicates little more than a necessity for more research into this relationship, isolating the effect of the primary independent variable (language fractionalization) onto GDP per capita.
