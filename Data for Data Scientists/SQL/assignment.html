The goal is to separate the fields in the JSON documents containing tweet and user information into two tables of a relational database. You can name these tables tweets and users. Before you write the tables into your database, you can process the tweet data with dplyr or other packages. Make sure that there are no duplicates among the users and tweets in the final database (i.e. the tweet id is unique in the tweets table, and the user id is unique in users table). Also make sure that these two tables can be combined afterwards using a column common to the tables. Please also add one indicator column (i.e. either TRUE/FALSE or 1/0) to each of your tables, in detail:

  • A column “screen_name_in” in the users table indicating whether an account is among the politician/party accounts
  • A column “in_reply_to_screen_name_in” in the tweets table indicating whether a tweet was a reply to a politician/party account

# Create a vector with all files to be imported with readtext
tweet_files <- list.files("~/Downloads/ge2017/", full.names=TRUE)
set.seed(20191210)

# Initialise the dataframe
data <- readtext(tweet_files[1], text_field = "texts", source = "twitter")

# Add all tweets to the dataframe
for (file in tweet_files[-1]){
  
  #message(file)
  # read next file of tweets into memory
  data_next <- readtext(file, source = "twitter")
  
  # Append the new dataframe to the already existing one
  data <- rbind(data, data_next)
}

# Remove unnecessary residual
rm(data_next)

# Separate 'data' data frame into 'user' and 'tweets' data frames, include the user id to make the two tables joinable in SQL
tweets <- data[,c(18, 1:14, 41, 42)]
users <- data[,c(18, 15, 16, 17, 19:40, 43, 44)]

# Read in the csv file
candidate_information <- read.csv("candidate_information.csv")

# Create new binary column "screen_name_in" in the `users` table indicating whether an account is among the politician/party accounts
users$screen_name_in <- ifelse(users$screen_name %in% candidate_information$screenName,1,0)

# Create a new binary column "in_reply_to_screen_name_in" in the `tweets` table indicating whether a tweet was a reply to a politician/party account
tweets$in_reply_to_screen_name_in <- ifelse(tweets$in_reply_to_screen_name %in% candidate_information$screenName,1,0)

# Convert readtext objects to dataframes
tweets <- as.data.frame(tweets)
users <- as.data.frame(users)

# Retain only unique values from tweets and users
tweets <- tweets[!duplicated(tweets$id_str), ]
users <- users[!duplicated(users$user_id_str), ]

# Convert created_at and user_created to datetime and back to text to make it possible to query
tweets$created_at <- parse_date_time(tweets$created_at, "%a %B %d %H:%M:%S %z %Y")
tweets$created_at <- as.character(tweets$created_at)

# Connect to SQL database
db <- dbConnect(RSQLite::SQLite(), "assignment-db.sqlite")
  
# adding both table to a SQL database
dbWriteTable(db, "users", users, append=TRUE)
dbWriteTable(db, "tweets", tweets, append=TRUE)
